## Spacing Scale

Declared values (must be multiples of 4). All values derived from existing Tailwind class usage in `Sidebar.tsx`, `EntityRow.tsx`, `EmptyState.tsx`, `SidebarNavItem.tsx`:

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon-to-label gaps, alert dot offsets |
| sm | 8px | `gap-2` / `px-2` / `py-2` | Entity row vertical padding, compact gaps |
| md | 16px | `px-4` / `gap-4` | Entity row horizontal padding, section gaps |
| lg | 24px | `gap-6` / `space-y-4` | Settings card spacing |
| xl | 32px | — | Layout column gaps (future) |
| 2xl | 48px | — | Major section breaks (future) |
| 3xl | 64px | `py-16` | EmptyState vertical centering |

Exceptions:
- Sidebar nav items use `px-3 py-2` (12px / 8px) matching existing `SidebarNavItem` — do not change to `px-4`.
- Touch target minimum: 44px height on `@media (pointer: coarse)` — already declared globally in `index.css`. No per-component override needed.
- Sidebar width: fixed 240px (`w-60`) — do not change.

## Typography

All values measured from existing class usage. Body text is 13px in sidebar/nav contexts; 14px in content areas.

Note: Sizes 12px, 13px, and 14px are each 1px apart — this reflects the existing codebase baseline. No new type sizes are introduced in Phase 1; the close steps are inherited, not designed.

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|----------|-------|
| Sidebar label | 13px | 500 (medium) | 1.5 | `text-[13px] font-medium` | All `SidebarNavItem` labels, "New Issue" button |
| Body / row text | 14px | 400 (regular) | 1.5 | `text-sm` | `EntityRow` title, form labels, settings paragraphs |
| Caption / meta | 12px | 400 (regular) | 1.3 | `text-xs` | `EntityRow` identifier (monospace), dates, section headers |
| Section heading | 12px | 500 (medium) | 1.3 | `text-xs font-medium uppercase tracking-wide` | Settings section headers (e.g. "Invites") |

Weights used: **regular (400)** and **medium (500)**. No bold weight introduced in this phase.

Monospace exception: issue identifiers use `font-mono` at `text-xs` — this is the only monospace context in Phase 1.

Invite URL display: read-only `<input>` at `text-sm font-mono` (mirrors the existing OpenClaw snippet textarea pattern in `CompanySettings.tsx`).

## Color

All values are CSS variables resolved from `ui/src/index.css`. Do not use raw oklch values in component code — reference variables only.

| Role | CSS Variable | Light value | Dark value | Usage |
|------|-------------|-------------|------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` (white) | `oklch(0.145 0 0)` (near black) | Page background, sidebar background |
| Secondary (30%) | `--sidebar` / `--muted` | `oklch(0.985 0 0)` / `oklch(0.97 0 0)` | `oklch(0.145 0 0)` / `oklch(0.269 0 0)` | Sidebar surface, muted backgrounds, hover states |
| Accent (10%) | `--accent` | `oklch(0.97 0 0)` | `oklch(0.269 0 0)` | Active nav item background (`bg-accent`), hover state (`bg-accent/50`), EmptyState icon container (`bg-muted/50`) |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` (red) | `oklch(0.637 0.237 25.331)` (red) | Error text (`text-destructive`), danger badge on sidebar |

Accent reserved for:
1. Active `SidebarNavItem` background (`bg-accent`)
2. Hover state on `SidebarNavItem` and `EntityRow` (`bg-accent/50`)
3. EmptyState icon container background (`bg-muted/50`)

Do NOT use accent for:
- Button backgrounds (use `--primary`)
- Badges (use `--primary` for default badge, `bg-red-600/90` for danger badge)
- Borders (use `--border`)

Badge colors (existing `SidebarNavItem` pattern — reuse exactly):
- Default numeric badge: `bg-primary text-primary-foreground`
- Danger badge (failed runs): `bg-red-600/90 text-red-50`

The `myTasks` badge on the new "My Tasks" sidebar item uses the **default** tone (`bg-primary text-primary-foreground`). It is not a danger indicator.

## Surface Specifications

### 1. My Tasks Page (`/my-tasks`)

Layout: matches existing `MyIssues.tsx` structure exactly. No layout changes.

Primary visual anchor (populated state): the issue list's first `EntityRow` — the task title is the focal point of each row.

```
┌─────────────────────────────────────┐
│ [PageSkeleton while loading]        │
│ OR                                  │
│ [EmptyState if 0 tasks]             │
│ OR                                  │
│ ┌─────────────────────────────────┐ │
│ │ border border-border            │ │
│ │ ┌─ EntityRow ─────────────────┐ │ │
│ │ │ StatusIcon  ID  Title  Date │ │ │
│ │ └─────────────────────────────┘ │ │
│ │ ┌─ EntityRow ─────────────────┐ │ │
│ │ │ ...                         │ │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

Page title (breadcrumb): "My Tasks" (change from current "My Issues").

### 2. My Tasks Sidebar Nav Item

Placed inside the "Work" `SidebarSection`, immediately after the "Issues" nav item.

```tsx
<SidebarNavItem
  to="/my-tasks"
  label="My Tasks"
  icon={ListTodo}
  badge={badges?.myTasks}
/>
```

- Badge is the `myTasks` count from `sidebarBadgesApi`.
- Badge only renders when `> 0` (existing `SidebarNavItem` behavior).
- Badge tone: `"default"` (no danger tone).

### 3. Human Invite Section in CompanySettings

Appended below the existing "Invites" section (after the OpenClaw invite block). Follows the exact same visual pattern as the existing OpenClaw invite block.

Structure:
```
┌─ Section card: border border-border rounded-md px-4 py-4 ─────┐
│  [text-xs text-muted-foreground]                               │
│  Invite a human member to this company.                        │
│                                                                │
│  [Button size="sm"]  Generate Human Invite Link               │
│                                                                │
│  [If URL generated]                                            │
│  ┌─ bg-muted/30 rounded-md border border-border p-2 ─────────┐│
│  │  [text-xs text-muted-foreground] Human Invite Link         ││
│  │  [Input readOnly value={inviteUrl} text-sm font-mono]      ││
│  │  [Button size="sm" variant="ghost"] Copy Link              ││
│  └────────────────────────────────────────────────────────────┘│
└────────────────────────────────────────────────────────────────┘
```

No dialog or modal — inline display only, matching the existing OpenClaw snippet pattern.

## Interaction States

### My Tasks list

| State | Visual |
|-------|--------|
| Loading | `<PageSkeleton variant="list" />` — full page |
| Empty (0 tasks) | `<EmptyState icon={ListTodo} message="No tasks assigned to you." />` |
| Error | `<p className="text-sm text-destructive">{error.message}</p>` above list |
| Populated | `EntityRow` list inside `border border-border` wrapper |
| Row hover | `bg-accent/50` (handled by `EntityRow` isClickable logic) |
| Row active (nav) | Row navigates to `/issues/:identifier` |

### Sidebar badge

| State | Visual |
|-------|--------|
| 0 tasks | Badge hidden (SidebarNavItem renders nothing when `badge == null || badge == 0`) |
| N > 0 tasks | `bg-primary text-primary-foreground` pill showing N |

### Human invite button

| State | Visual |
|-------|--------|
| Idle | `Button size="sm"` with label "Generate Human Invite Link" |
| Pending | Button `disabled`, label "Generating..." |
| Success | URL display block appears inline; button returns to idle |
| Error | `text-sm text-destructive` paragraph below button: "Failed to generate invite link. Please try again or refresh the page." |
| URL copied | "Copied" with `Check` icon in green, disappears after 2s |
