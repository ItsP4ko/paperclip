<important>
COOKIE CACHE CLARIFICATION: cookieCache is NOT configured in this project; session revocation
is immediate per RESEARCH.md Pitfall 3. The "60s TTL" mentioned in the phase success criterion
(SC3) is incorrect -- treat revocation as immediate. When a session is revoked via
`/api/auth/revoke-session`, BetterAuth deletes the session row from the database immediately.
The next `getSession()` call for that token returns null with zero delay.

CURRENT SESSION IDENTIFICATION (CRITICAL): The custom `GET /api/auth/get-session` handler in
`server/src/app.ts` (line 147) returns a SYNTHETIC session ID (`paperclip:<source>:<userId>`),
NOT the real BetterAuth session UUID. BetterAuth's `listSessions()` returns real UUID session IDs.
These will NEVER match. Therefore, do NOT use `authApi.getSession()` to identify the current session.

Instead, identify the current session by comparing each session's `token` field from `listSessions()`
against the bearer token stored in `localStorage.getItem("paperclip_session_token")`. This token
is set during sign-in (see `ui/src/api/auth.ts` line 50: `localStorage.setItem("paperclip_session_token", authToken)`)
and is the actual BetterAuth session token value that matches the `token` field in `listSessions()`.
</important>

Add three new methods to the `authApi` object:

```typescript
listSessions: async (): Promise<SessionEntry[]> => {
  const res = await fetch(`${API_BASE}/auth/list-sessions`, {
    credentials: "include",
    headers: { Accept: "application/json", ...getBearerHeaders() },
  });
  if (res.status === 401) { handle401(); return []; }
  if (!res.ok) throw new Error(`Failed to list sessions (${res.status})`);
  return res.json();
},

revokeSession: async (token: string): Promise<void> => {
  const res = await fetch(`${API_BASE}/auth/revoke-session`, {
    method: "POST",
    credentials: "include",
    headers: { "Content-Type": "application/json", ...getBearerHeaders() },
    body: JSON.stringify({ token }),
  });
  if (res.status === 401) { handle401(); return; }
  if (!res.ok) throw new Error(`Failed to revoke session (${res.status})`);
},

revokeOtherSessions: async (): Promise<void> => {
  const res = await fetch(`${API_BASE}/auth/revoke-other-sessions`, {
    method: "POST",
    credentials: "include",
    headers: { "Content-Type": "application/json", ...getBearerHeaders() },
    body: JSON.stringify({}),
  });
  if (res.status === 401) { handle401(); return; }
  if (!res.ok) throw new Error(`Failed to revoke other sessions (${res.status})`);
},
```

Also add a helper to get the current session token for matching:
```typescript
getCurrentSessionToken: (): string | null => {
  try {
    return localStorage.getItem("paperclip_session_token");
  } catch {
    return null;
  }
},
```
