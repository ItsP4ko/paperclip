  <files>vercel.json</files>
  <read_first>
    - vercel.json (current state with Content-Security-Policy-Report-Only header)
  </read_first>
  <action>
    In `vercel.json`, change the CSP header key from report-only to enforcing. This is a single string replacement:

    Change:
    ```json
    "key": "Content-Security-Policy-Report-Only",
    ```

    To:
    ```json
    "key": "Content-Security-Policy",
    ```

    The `value` string (the CSP directive) MUST remain exactly the same -- do not change any directives. Only the key name changes.

    The resulting header entry should be:
    ```json
    {
      "key": "Content-Security-Policy",
      "value": "default-src 'self'; script-src 'self' 'sha256-2oMWziIOZ3clEgoBlgQGv6iO60JrWMqbpz+42WnDniA='; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob:; font-src 'self' data:; connect-src 'self' https://paperclip-paperclip-api.qiwa34.easypanel.host wss://paperclip-paperclip-api.qiwa34.easypanel.host; worker-src 'self'; frame-ancestors 'none'; base-uri 'self'; form-action 'self';"
    }
    ```
  </action>
  <verify>
    <automated>node -e "const j = require('./vercel.json'); const h = j.headers?.[0]?.headers?.[0]; if (!h) { console.error('No header found'); process.exit(1); } if (h.key !== 'Content-Security-Policy') { console.error('Header key is: ' + h.key + ' (expected Content-Security-Policy, not Report-Only)'); process.exit(1); } if (h.key === 'Content-Security-Policy-Report-Only') { console.error('Still in report-only mode'); process.exit(1); } console.log('CSP header promoted to enforcing'); process.exit(0);"</automated>
  </verify>
  <acceptance_criteria>
    - vercel.json contains `"key": "Content-Security-Policy"` (NOT `Content-Security-Policy-Report-Only`)
    - vercel.json does NOT contain the string `Content-Security-Policy-Report-Only`
    - The CSP directive value string is unchanged from Plan 01
    - vercel.json is valid JSON (node -e "require('./vercel.json')" exits 0)
