## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`), inline badge padding (`px-1`) |
| sm | 8px | Compact row padding (`py-2`, `px-2`) |
| md | 16px | Default section padding (`px-4`, `py-4`), card inner padding |
| lg | 24px | Section breaks inside IssueDetail panel (`space-y-6` = 24px) |
| xl | 32px | Page-level vertical spacing |
| 2xl | 48px | EmptyState top/bottom padding (`py-16` = 64px â€” see exception) |
| 3xl | 64px | Reserved; not used in this phase |

**Exceptions:**
- EmptyState uses `py-16` (64px) â€” existing component; do not change.
- Touch targets: coarse-pointer devices enforce `min-height: 44px` globally via `index.css` media query. All interactive elements in the human action bar must meet this automatically.
- IssueProperties property rows retain their existing `py-1.5` styling; no changes are made to this component in Phase 2.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | Default prose, property labels, list item text |
| Caption | 12px (`text-xs`) | 400 (`font-normal`) | 1.4 | Timestamps, muted metadata, filter section labels |
| Label | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | Section headings in sidebar panels, button text, action bar headings |
| Display | 20px (`text-xl`) | 700 (`font-bold`) | 1.2 | Issue title (`text-xl font-bold` â€” verified IssueDetail:1304) |

**Declared weights:** 400 (`font-normal`) and 700 (`font-bold`) â€” exactly two.
Use 400 for all body copy, labels, captions, and action bar headings.
Use 700 only for the issue title, matching the existing IssueDetail pattern.

**Primary visual anchor:** The issue title (`text-xl font-bold`) is the focal point of the IssueDetail view. The eye must land here first before scanning the HumanActionBar and the property sidebar. No other element in the view uses `text-xl font-bold`.

**Editor / content body:** `0.875rem` (14px) at `line-height: 1.5` â€” from MDXEditor theme
integration in `index.css`. The `paperclip-edit-in-place-content` class uses `0.9375rem`
(15px) at `line-height: 1.75rem` â€” use this for inline-editable title fields only.

## Color

All values are CSS custom properties defined in `index.css`. Use the token names,
not raw hex/oklch values, so dark mode works automatically.

| Role | Token | Light value (approx) | Usage |
|------|-------|----------------------|-------|
| Dominant (60%) | `--background` | white | Page surfaces, IssueDetail main area |
| Secondary (30%) | `--sidebar` / `--card` / `--muted` | near-white gray | Sidebar, property panel, tab strip, skeleton loaders |
| Accent (10%) | `--accent` | light gray (`oklch(0.97 0 0)`) | See reserved-for list below |
| Destructive | `--destructive` | red (`oklch(0.577 0.245 27.325)`) | Destructive actions only |
| Primary | `--primary` | near-black | Primary buttons, active status indicators |
| Muted foreground | `--muted-foreground` | medium gray | Secondary labels, placeholder text, timestamps |

**Accent reserved for:**
- Active/selected state on filter toggle buttons (toggle is active: `bg-accent`)
- Hover background on list rows and property picker rows (`hover:bg-accent/50`, `hover:bg-accent/20`)
- Active tab background in TabsList
- Highlighted activity rows (via `dashboard-activity-highlight` animation)
- Status badge and label chip backgrounds where color is decorative (e.g. `bg-cyan-500/10` uses colored accents for AI-run badge, not the `--accent` token)

**Destructive usage in this phase:**
- Permission error state in IssueDetail (`border-destructive/30 bg-destructive/10 text-destructive` â€” existing pattern at line 1160)
- Reassignment warning dialog destructive border (`border-destructive/30 bg-destructive/5`)
- "Remove" / "Delete" dropdown item text (`text-destructive` â€” existing pattern at line 1283)
- Server-side 403/422 error toast text

**Status colors (special â€” not from 60/30/10 split):**
Handled by `issueStatusIcon` in `lib/status-colors.ts`. Use existing `StatusIcon` component.
Do not introduce new status color tokens; extend the existing map only.

### HumanActionBar layout contract

Shown only when `issue.assigneeUserId === currentUserId`. Positioned inside IssueDetail
**above the comment/tabs section**, below the issue metadata row.

```
[ Status: [StatusIcon + label dropdown] ]   [ Attach File [icon button] ]   [ Add Subtask [icon button] ]
```

- Container: `flex items-center gap-2 py-2 border-t border-border`
- Status control: reuse `<StatusIcon showLabel onChange={...} />` â€” already renders Popover
- Attach file: `Button` variant ghost size sm with `Paperclip` lucide icon + "Attach file" label (hidden on mobile, icon-only with Tooltip)
- Add subtask: `Button` variant ghost size sm with `Plus` lucide icon + "Add subtask" label (hidden on mobile, icon-only with Tooltip)
- Touch target minimum: 44px height met by global CSS rule

### ReassignWarningDialog copy and layout

Triggered when: `issue.assigneeAgentId !== null && issue.status === "in_progress"` AND user selects a `user:*` value in the assignee picker.

- Uses `Dialog` from `ui/components/ui/dialog.tsx` â€” same structure as `ExecutionWorkspaceCloseDialog.tsx`
- DialogTitle: "Interrupt AI task?"
- DialogDescription: "This task is currently running with [agent name]. Reassigning it to you will interrupt the active run. The agent's progress will remain visible in the activity log."
- Confirm button: `variant="destructive"` label "Reassign to me"
- Cancel button: `variant="outline"` label "Keep AI assigned"

### Assigned-to-me toggle button

Added to the IssuesList filter toolbar. Matches existing quick-filter pill style at line 443â€“446:

```
className="px-2 py-1 text-xs rounded-full border transition-colors {active: bg-primary text-primary-foreground border-primary} {inactive: border-border text-muted-foreground hover:text-foreground hover:border-foreground/30}"
```

Icon: `<User className="h-3.5 w-3.5 mr-1" />` inline before label text "Assigned to me".

## Interaction States

### Status change (ACTN-01)

| State | Visual |
|-------|--------|
| Default | StatusIcon colored circle with status color from `issueStatusIcon` map |
| Hover | cursor-pointer, StatusIcon circle gets `hover:opacity-80` |
| Popover open | Popover shows all statuses as ghost Buttons; active status has `bg-accent` |
| Loading (mutation pending) | StatusIcon shows skeleton pulse; use TanStack Query `isPending` |
| Error | Toast notification â€” "Failed to update status. Try again." |

### File attachment (ACTN-02)

| State | Visual |
|-------|--------|
| Idle | Ghost button with Paperclip icon |
| Drag over | `bg-accent/20 border-dashed border-2 border-border` on drop zone |
| Uploading | Button disabled, spinner icon replaces Paperclip icon |
| Error | Inline text `text-xs text-destructive` below attachments list |

### Subtask creation (ACTN-03)

| State | Visual |
|-------|--------|
| Trigger | Click "Add subtask" button opens inline Input below subtask list |
| Input | `Input` component, placeholder "Subtask title", `onKeyDown Enter` submits |
| Pending | Input disabled, spinner |
| Success | Input clears, new subtask row appears with `status: "todo"` |
| Error | Input border turns destructive, inline error text below |

### Reassign warning dialog (ASGN-03)

| State | Visual |
|-------|--------|
| Trigger | Assignee picker selects `user:*` while AI run is active |
| Dialog open | Standard Radix Dialog with dark overlay, `max-w-md` |
| Confirm | Dialog closes, mutation fires with `resolveAssigneePatch(value)` |
| Cancel | Dialog closes, assignee picker reverts to previous value |

### "Assigned to me" toggle (TASKS-03)

| State | Visual |
|-------|--------|
| Inactive | `border-border text-muted-foreground` pill |
| Active | `bg-primary text-primary-foreground border-primary` pill |
| Toggle | Instant client-side filter; no loading state needed |

### Permission error (PERM-01)

| State | Visual |
|-------|--------|
| 403 from server | Toast notification â€” "You can only edit tasks assigned to you." |
| Optimistic UI rollback | Mutation's `onError` reverts to previous issue state via TanStack Query |

## Accessibility Notes

- All interactive elements in the human action bar must have accessible labels (`aria-label` or visible text).
- Tooltip wraps icon-only buttons (`Attach file`, `Add subtask`) on desktop to supply accessible names.
- ReassignWarningDialog uses Radix Dialog â€” focus trap and `aria-modal` handled automatically.
- Status picker Popover uses Radix Popover â€” keyboard navigation handled automatically.
- "Assigned to me" toggle pill is a `<button>` element with `aria-pressed` reflecting active state.
- Minimum touch target 44px is globally enforced by `index.css` `@media (pointer: coarse)` rule.

## Dark Mode

All color tokens use CSS custom properties. Both light (`:root`) and dark (`.dark`) values
are defined in `index.css`. No additional dark-mode overrides are needed for Phase 2
components â€” use semantic token names exclusively (`bg-accent`, `text-destructive`,
`border-border`, etc.).

The `dashboard-activity-highlight` animation uses `var(--primary)` and `var(--accent)` and
works in both modes automatically.
