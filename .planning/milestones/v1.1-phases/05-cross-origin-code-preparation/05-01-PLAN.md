<objective>
Wire the Express backend for cross-origin operation: CORS middleware, BetterAuth cookie SameSite override, secret fallback removal, and boardMutationGuard external origin support.

Purpose: Without these server-side changes, a Vercel-hosted frontend cannot authenticate or mutate data on a Railway-hosted backend. CORS blocks preflight, cookies get dropped (SameSite=Lax default), and the mutation guard rejects all cross-origin POSTs.

Output: Backend fully configured for cross-origin credentialed requests from an allowed Vercel origin, with tests proving each behavior.
</objective>

    2. Inside `createApp()`, BEFORE the existing `app.use(express.json({...}))` block (before line 89), insert:

    ```typescript
      `https://${h}`,
      `http://${h}`,
    ]);

    app.use(
      cors({
        origin: (origin, callback) => {
          if (!origin || corsAllowedOrigins.includes(origin)) {
            callback(null, true);
          } else {
            callback(new Error("Not allowed by CORS"));
          }
        },
        credentials: true,
      }),
    );
    ```
