The user must manually apply all 49 migration SQL files to the Supabase database. Per the project rule (CLAUDE.md): "NUNCA ejecutar migraciones automaticamente."

IMPORTANT: Apply to a COMPLETELY EMPTY Supabase database. If the database has pre-existing tables but no Drizzle migration journal, the server will crash with "Database has tables but no migration journal."

**Procedure:**

Claude will prepare the migration SQL for the user by concatenating all 49 files. The executor should:

1. Read all 49 migration files from `packages/db/src/migrations/` in order (0000 through 0048)
2. Concatenate them into a single SQL script, separating each file's content with a comment header like:
   ```sql
   -- Migration: 0000_fluffy_the_captain.sql
   [contents]
   -- Migration: 0001_next_migration.sql
   [contents]
   ```
3. Present the full SQL to the user

**Steps for the user:**

4. Go to your Supabase project dashboard
5. Click **SQL Editor** in the left sidebar
6. Create a new query
7. Paste the concatenated SQL (or paste file-by-file if preferred)
8. Click **Run** to execute
9. Verify success: run this query in the SQL Editor:
   ```sql
   SELECT count(*)::int AS table_count
   FROM information_schema.tables
   WHERE table_schema = 'public'
     AND table_type = 'BASE TABLE';
   ```
   Expected result: approximately 40+ tables.

10. Verify migration journal exists:
    ```sql
    SELECT count(*)::int AS migration_count
    FROM drizzle.__drizzle_migrations;
    ```
    Expected result: 49 rows (one per migration file).

NOTE: The server's `applyPendingMigrations()` function will detect the existing journal on boot and skip re-application. This is the safe path.

ALTERNATIVE (simpler approach): Instead of concatenating manually, the user can run the server locally pointed at the Supabase URL and let it auto-migrate the empty DB. The server's `applyPendingMigrations()` handles fresh empty databases via `migratePg()`. Run:
```bash
DATABASE_URL="postgresql://postgres.PROJECT_REF:[PASS]@aws-0-REGION.pooler.supabase.com:5432/postgres" \
PAPERCLIP_DEPLOYMENT_MODE=authenticated \
BETTER_AUTH_SECRET=$(openssl rand -hex 32) \
node --import ./server/node_modules/tsx/dist/loader.mjs server/dist/index.js
```
Wait for "Server listening" then Ctrl+C. The migration journal will be created. Verify with the SQL queries above.
