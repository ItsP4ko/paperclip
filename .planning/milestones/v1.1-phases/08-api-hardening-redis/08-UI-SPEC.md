# Phase 8 — UI Design Contract

> Visual and interaction contract for Phase 8: API Hardening & Redis.
> Generated by gsd-ui-researcher.

---

## Phase UI Impact Assessment

**This phase is backend-only.** Phase 8 adds Express middleware (helmet, rate limiting) and a Redis
cache layer. All six requirements (HARD-01, HARD-02, HARD-03, REDIS-01, REDIS-02, REDIS-03) produce
no new pages, no new routes, and no new components.

The only frontend-adjacent surface is `GET /api/instance/settings/general`, which gains a Redis
cache and a `PATCH` handler that invalidates that cache. Both operations are transparent to the
frontend — the existing `InstanceGeneralSettings` page at `ui/src/pages/InstanceGeneralSettings.tsx`
requires no changes.

One new user-visible state is introduced: a `429 Too Many Requests` error response when the rate
limiter threshold is exceeded. The existing error-handling pattern in the app (inline
`text-sm text-destructive` or `rounded-md border border-destructive/40 bg-destructive/5` error
banners) covers this without any new UI work.

The design contracts below are therefore *continuation contracts* — they document the existing
system tokens and patterns that backend error responses must be consistent with, and that any
future UI work in this phase would inherit. No new components are commissioned.

## Spacing Scale

Standard 8-point scale in use across the codebase (source: `design-guide` skill, `InstanceGeneralSettings.tsx`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `p-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `p-4`) |
| lg | 24px | Section padding (`p-6`, `space-y-6`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets on coarse-pointer devices: `min-height: 44px` (enforced via `@media (pointer: coarse)` in `index.css`)
- Toggle switch thumb: `h-3.5 w-3.5` (intentionally below scale — this is a fixed UI component size, not a spacing token)

## Color

All tokens are CSS variables in OKLCH space, defined in `ui/src/index.css`. Use semantic names only.

| Role | Token | Light Value | Dark Value | Usage |
|------|-------|-------------|------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` (white) | `oklch(0.145 0 0)` (near-black) | Page background, surfaces |
| Secondary (30%) | `--card` / `--muted` | `oklch(1 0 0)` / `oklch(0.97 0 0)` | `oklch(0.205 0 0)` / `oklch(0.269 0 0)` | Cards, sidebar, nav, muted backgrounds |
| Accent (10%) | `--accent` | `oklch(0.97 0 0)` | `oklch(0.269 0 0)` | Hover states on rows and nav items only |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` (red) | `oklch(0.637 0.237 25.331)` (red) | Error banners, destructive action buttons |

**Accent reserved for:** hover states on entity rows (`hover:bg-accent/50`), active nav items, and selected option buttons. Never used as a general highlight color.

**Rate limit error (429) color treatment:** Use the existing destructive error banner pattern —
`rounded-md border border-destructive/40 bg-destructive/5 px-3 py-2 text-sm text-destructive`.
No new color tokens are introduced.

## Copywriting Contract

Phase 8 introduces one new user-visible error state (HTTP 429) and no new UI surfaces. The
copywriting contract covers this error and confirms existing patterns for the cached settings page.

| Element | Copy | Notes |
|---------|------|-------|
| Rate limit error (429) | "Too many requests. Please slow down." | Source: `08-RESEARCH.md` Pattern 3 — rate limiter `handler` response body. Used in API JSON response only, not surfaced as a UI string directly. Frontend displays generic "Request failed: 429" unless the app parses the body. |
| Settings load error | "Failed to load general settings." | Pre-existing — `InstanceGeneralSettings.tsx` line 97. No change needed. |
| Settings update error | "Failed to update general settings." | Pre-existing — `InstanceGeneralSettings.tsx` line 44. No change needed. |
| Empty state | Not applicable | Phase 8 introduces no list views. |
| Primary CTA | Not applicable | Phase 8 introduces no new user actions. |
| Destructive confirmation | Not applicable | Phase 8 introduces no destructive user-facing actions. The "Disable All" timer heartbeat action in `InstanceSettings.tsx` pre-exists and is out of scope. |

**If the frontend does choose to display the rate limit error body as a toast or banner**, use:
- Heading: "Request throttled"
- Body: "You've sent too many requests. Wait a moment and try again."
- Pattern: existing destructive inline banner — `border-destructive/40 bg-destructive/5 text-destructive`
