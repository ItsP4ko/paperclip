### Integration Issues

**INT-01: Members endpoint permission gap (authenticated mode)**
`GET /companies/:id/members` requires `users:manage_permissions` grant. Human members joining via invite receive zero grants (no `defaultsPayload.human.grants`). In authenticated mode, non-owner human members get 403 — assignee pickers silently show empty "Team Members" sections. Owner path works because bootstrap CEO has full grants.
- File: `server/src/routes/access.ts:2933`
- Affects: ASGN-01, ASGN-02, IDENT-03 (degraded, not broken)

**INT-02: resolveAssigneeName orphaned**
Exported at `ui/src/lib/assignees.ts:99`, imported only by test file. Production components use inline `m.userDisplayName ?? m.userEmail ?? m.principalId.slice(0, 8)` instead.
- Affects: IDENT-03, IDENT-04 (cosmetic — data flows correctly)

**INT-03: TypeScript compile error**
`ui/src/components/IssueProperties.tsx:227` — `AssigneeSelection` not assignable to `Record<string, unknown>`. Runtime correct, `tsc --noEmit` fails.
- Affects: ASGN-03, ACTN-04, ACTN-05 (build-only)
