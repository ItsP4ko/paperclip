### Integration Notes

**Dual heartbeat mechanisms (non-blocking):** Two independent dead-connection mechanisms coexist:
1. Server-side (WS protocol): 30s ping frames from `live-events-ws.ts`
2. Client-side (application): 10s interval + 12s deadline JSON pings from `LiveUpdatesProvider.tsx`

The server has no `onmessage` handler for client JSON `{ type: "ping" }` frames. Client heartbeat will always time out after 22s of server silence, even in healthy idle sessions. This satisfies WS-01 (≤25s) but causes unnecessary reconnect churn when no live events arrive. Future fix: add server-side echo of client ping to prevent spurious reconnects.
